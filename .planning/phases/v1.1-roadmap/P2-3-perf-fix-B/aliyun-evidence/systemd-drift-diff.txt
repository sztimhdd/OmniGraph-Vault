> # OmniGraph KB v2 — FastAPI service, systemd unit reference
> #
> # Authored by phase kb-v2.1-1 (KG mode hardening, 2026-05-15).
> #
> # This file is a REFERENCE template, not an auto-installed unit. Aliyun and
> # any other production operator copies/edits this to the host's
> # /etc/systemd/system/kb-api.service then runs `systemctl daemon-reload`
> # + `systemctl restart kb-api.service`.
> #
> # Memory profile rationale (Aliyun ECS, 3.4Gi total RAM):
> #   - vitaclaw-site Node process holds ~500MB
> #   - kb-api uvicorn + LightRAG + embedding cache + Python runtime: typical
> #     800MB-1.4GB during a KG search; spike up to 1.8GB observed when an
> #     embedding batch hits Vertex AI in parallel with FTS5 query traffic
> #   - MemoryHigh=1.5G triggers cgroup throttling before OOM (graceful slowdown)
> #   - MemoryMax=2G is the hard cap; kernel kills at this; systemd auto-restarts
> #   - StartLimitBurst=5 / IntervalSec=60 prevents OOM-restart-OOM-restart loops
> #     from monopolizing the host (5 restarts in 60s pauses the unit)
> #
> # Adjust ExecStart and Environment paths to match your deploy layout.
> # Recommended: keep KB_KG_GCP_SA_KEY_PATH OFF / unset on hosts where the GCP
> # SA JSON is not present — the kb-api will boot in controlled-degraded mode
> # (FTS5 search + /api/synthesize fts5_fallback continue to work) instead of
> # failing on every KG request.
