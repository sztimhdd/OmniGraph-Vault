## Atomic commit strategy

To stay within Surgical Changes principle (#3), the work lands in **clusters**, each as a single atomic commit:

1. **Deletes cluster** — remove all DELETE-verdict items in one commit, with PR-style summary listing every removed file/function and rationale (cite P4-AUDIT.md row IDs). Land first because it shrinks subsequent diff scope.
2. **Rewrites cluster** — REWRITE-verdict items, one logical commit per coherent rewrite (e.g., "REWRITE Reasoner prompts for 1M-context", "REWRITE tool definitions for current Claude tool-use schema").
3. **Wire-to-frontend cluster** — new "Deep Research" tab + connecting glue. Single commit covering UI + endpoint + integration test.

Each cluster lands behind a deploy-parity gate (Aliyun + Databricks both verified before next cluster starts).
