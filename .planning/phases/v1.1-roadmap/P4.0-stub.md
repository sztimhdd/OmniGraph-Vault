## Audit criterion (the deciding question for every item)

> *"Was this code authored to work around the 2025 200k-context-window era? In the 2026 1M-context environment, does it still earn its keep?"*

Reference [Anthropic 2026 doubled limits](https://www.dotzlaw.com/insights/anthropic-2026-code-with-claude/): *"When the platform absorbs your summarization scaffolding, the value of having written a clever summarizer drops to zero. Audit your harness for components whose only job is to compensate for the context window being too small."*

Verdict criteria:
- **DELETE**: code whose only purpose was fighting context-window pressure (per-turn summarizers, aggressive context compression, fragmenting reasoning into smaller LLM calls because one big call wouldn't fit)
- **KEEP**: code that delivers user value independent of context size (tool-use orchestration, agentic decomposition of multi-step research questions, source citation, eval scaffolding)
- **REWRITE**: code with sound intent but built for an obsolete substrate (e.g., assumes specific LLM provider, hardcodes 200k limits, wraps deprecated APIs)
- **WIRE-TO-FRONTEND**: code that is ready and just needs UI surface (ARAG's user-visible deep-research surface)
