## Why partial-pass closes v1.0.y-0

v1.0.y-0's stated goal (plan §"Goal") is:

> Land the single-line config drift fix that closes the P0 ghost-success
> pattern observed on Databricks long_form synthesize: outer
> asyncio.wait_for(timeout=60) preempts before LightRAG hybrid query can
> finish, returning markdown_len=0 + confidence=no_results +
> error="C1 timeout" at wallclock ~64s.

The post-fix evidence shows:

- The wait_for(60) preempt **no longer fires** (wallclock 98s, gate_b PASS)
- The `confidence=no_results` **no longer occurs** (now `confidence=kg`)
- The `error="C1 timeout"` **no longer occurs** (`error_field=None`)

Three of the four originally-cited symptoms are eliminated. Only
`markdown_len=0` survives, and it is now **causally distinct** from the C1
timeout — it persists with `confidence=kg + error=None`, which is impossible
under the old code path. The timeout drift is therefore closed; the
markdown emission is a different bug.
