## Goal

Restore preservation of `![alt](url)` markdown image references through the
ARAG (b-branch) Synthesizer → Reasoner → Retriever pipeline so that
LightRAG-retrieved chunks containing image markdown survive Reasoner
re-formulation and reach the user-facing synthesis output intact.

This phase **plan-authors now** (per Q1 default assumption — plan
authoring is NOT gated on the b-branch probe report). However, **task
execution IS gated** on the b-branch probe report landing in
`.scratch/`. The Task list contains a `<halt>` block that fires only
at execute-phase time if the probe report has not yet landed.

## Background

### Symptom

User-reported regression on the Agentic-RAG-v1 (b) branch: when a
LightRAG hybrid retrieval surfaces a chunk that contains
`![image-alt](http://localhost:8765/images/<hash>/<file>.png)`
markdown, the Reasoner stage's chunk-rewrite step strips the image
markdown, leaving only the alt text or nothing at all. By the time
the Retriever's output reaches the Synthesizer's final markdown
emission, the image refs are gone — final answer renders without
images.

This is functionally identical to the v1.0.x KOL ingestion `image_ref`
strip class but on the **read** side (synthesis pipeline) rather than
the **write** side (ingest pipeline). The defect surface is the
Reasoner's prompt and/or post-processing on retrieved chunks.

### Why this is non-trivial

The (b) branch Synthesizer→Reasoner→Retriever pipeline (per
`docs/design/agentic_rag_internal_api.md`) introduces two LLM hops
between the LightRAG `aquery()` call and the user-facing markdown:

1. **Retriever** — calls `omnigraph_search.query.search()` (the
   stable cross-milestone API contract); returns raw chunks with
   image markdown intact.
2. **Reasoner** — re-formulates retrieved chunks into a structured
   reasoning trace; this is the suspect strip site.
3. **Synthesizer** — composes final markdown from reasoning trace.

The image refs have to survive 2 LLM-mediated hops; either Reasoner's
prompt instructs the LLM to strip / paraphrase chunk content (LLM
follows; image markdown gone) or Reasoner's post-processing regex
strips structured artifacts that catch image markdown as a side
effect.
