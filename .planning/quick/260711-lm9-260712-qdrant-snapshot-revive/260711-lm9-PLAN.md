## Tasks

### T1 — Fix the timer structural defect
- **File:** `deploy/aliyun/systemd/qdrant-snapshot.timer`
- **Action:** Replace monotonic `OnBootSec=15min` + `OnUnitActiveSec=6h` with wall-clock
  `OnCalendar=*-*-* 03/6:10:00 UTC` (fires 03:10/09:10/15:10/21:10 UTC — same 4×/day cadence).
  Keep `Persistent=true`. Add a comment documenting the #78 root cause.
- **Why:** `OnUnitActiveSec` re-arms ONLY after a *successful* activation, so one failed boot-fire
  disarms the whole schedule permanently. `OnCalendar` fires on wall-clock regardless of prior
  outcome — a single failed run can no longer kill the schedule.
- **Verify:** `systemd-analyze calendar` parses; after SCP + daemon-reload + stop/start,
  `list-timers` shows a real NEXT.
