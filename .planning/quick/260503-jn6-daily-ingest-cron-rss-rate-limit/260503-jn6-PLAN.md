<objective>
Fix the daily-ingest cron's two ongoing bugs:

  Bug A (RSS gap): the current cron body
    `run batch_ingest_from_spider.py --from-db --topic-filter ... --min-depth 2`
  only ingests KOL articles. `enrichment/rss_ingest.py` is never called by any
  cron, so RSS is never ingested into LightRAG.

  Bug B (no rate limit): Day-1 backlog is 249 KOL depth>=2 + 479 RSS; a single
  cron fire would blow past reasonable per-run limits.

Fix strategy:
  - Call `orchestrate_daily.py --step 7` from the cron (step_7_ingest_all
    already runs KOL then RSS sequentially — wiring already exists).
  - Add `--max-kol N` + `--max-rss N` knobs so each cron fire is capped.
  - Thread `--max-articles` into `batch_ingest_from_spider.ingest_from_db()`
    (currently the flag exists but is silently ignored on the --from-db path).
  - Rewrite the daily-ingest cron body in `register_phase5_cron.sh` and add
    update-or-add semantics so re-running the register script actually swaps
    the body when it already exists.

Non-negotiables (from task description):
  - Other 5 crons unchanged (rss-fetch / rss-classify / daily-classify-kol /
    daily-enrich / daily-digest).
  - orchestrate_daily.py steps 1-6 and 8-9 behaviour unchanged.
  - D-07 REVISED + D-19: step_6_enrich_deep stays KOL-only forward-only.
  - LLM routing unchanged: DeepSeek for classify/translate, Gemini Vision+Embed only.
  - All unit tests mock-only (Cisco Umbrella proxy blocks real HTTP).

Purpose: close the daily-ingest RSS gap + make each cron fire rate-limited so

       replace_job() {
         local name="$1"
         local schedule="$2"
         local prompt="$3"

         if printf '%s\n' "$EXISTING" | grep -qE "\b${name}\b"; then
           echo "REPLACE ${name} — removing existing then re-adding"
           # Best-effort remove. If CLI subcommand differs, caller must
           # manually remove per SUMMARY § operator checklist.
           hermes cron remove --name "${name}" 2>/dev/null \
             || hermes cron delete --name "${name}" 2>/dev/null \
             || { echo "  (remove failed; see SUMMARY for manual steps)"; return 0; }
         else
           echo "ADD  ${name} @ ${schedule}"
         fi

         hermes cron add \
           --name "${name}" \
           --workdir "${OMNIGRAPH_ROOT:-$HOME/OmniGraph-Vault}" \
           "${schedule}" \
           "${prompt}"
       }
