## Wave 2 — skip_reason_version column (M, ~1 day)

### Goal
Add `skip_reason_version INTEGER NOT NULL DEFAULT 0` column to `ingestions` so candidate
SELECT can distinguish "skipped under current reject taxonomy (permanently dead URL,
don't retry)" from "skipped under prior taxonomy (re-evaluate when version bumps)".

### Why
Current candidate SELECT (`_build_topic_filter_query`, line 1395-1431 of
`batch_ingest_from_spider.py`) only excludes `status='ok'` rows. `status='skipped'` rows
get re-pulled every cron run — wastes Apify quota + Layer 1 LLM calls on permanently
dead URLs (404 / deleted / structural reject).

Reference: CLAUDE.md "Lessons Learned" 2026-05-05 #6.

### Design
- New column: `ingestions.skip_reason_version INTEGER NOT NULL DEFAULT 0`
  - 0 = legacy / unset (backfill applied to all existing rows during migration)
  - 1 = current code's reject taxonomy version
- New module-level constant in `batch_ingest_from_spider.py`:
  - `SKIP_REASON_VERSION_CURRENT = 1`
- Candidate SELECT anti-join changes (KOL branch line 1405-1408 + RSS branch line
  1422-1425): exclude `status='ok'` always; exclude `status='skipped' AND
  skip_reason_version = ?` (parameterized).
- Params tuple expands from 2-tuple `(PROMPT_VERSION_LAYER1, PROMPT_VERSION_LAYER1)` to
  4-tuple `(SKIP_REASON_VERSION_CURRENT, PROMPT_VERSION_LAYER1,
  SKIP_REASON_VERSION_CURRENT, PROMPT_VERSION_LAYER1)` — one binding per UNION branch
  for both predicates.
- All 7 `INSERT OR REPLACE INTO ingestions(article_id, source, status)` sites in
  `batch_ingest_from_spider.py` (lines 1566, 1691, 1742, 1787, 1799, 1817, 1841) extended
  to write `skip_reason_version` column with value `SKIP_REASON_VERSION_CURRENT`.

### Workflow this enables
Future taxonomy bump (new reject reason / refined Layer 1 prompt) → bump
`SKIP_REASON_VERSION_CURRENT` → old skipped rows (version stays 0 or N-1) re-enter
candidate pool because their version no longer matches `?`. Permanent-dead URLs that
got rejected under current version stay excluded forever (good).

### Scope
