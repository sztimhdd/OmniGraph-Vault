**Step 1 — Create 5 segment fetch functions:**

All functions take (org: str, repo: str) and return str | None. Each builds a focused markdown document with a clear heading. Use the existing _gh_api() helper throughout.

1. `_fetch_identity(org, repo) -> str | None`
   - Refactor from existing fetch_repo_content() lines 98-136.
   - Keep: metadata block (description, stars, language, topics, license, URL, default branch), top-level tree, and README.
   - This is the "identity card" — same content as today, just extracted into its own function.
   - Title: `# {org}/{repo} — Identity`

2. `_fetch_docs(org, repo) -> str | None`
   - Call `_gh_api(f"repos/{org}/{repo}/contents/docs")`.
   - _gh_api returns a **list** for directory listings (not a dict). If None or not a list, return None (docs/ doesn't exist — skip silently).
   - Filter to .md and .rst files only: `[f for f in listing if f["name"].endswith((".md", ".rst"))]`
   - For each file (cap at 10 files): call `_gh_api(f["url"])` to get file content, base64-decode `result["content"]`, append to doc.
   - Title: `# {org}/{repo} — Documentation`
   - Each file as `## {filename}\n\n{decoded_content}`

3. `_fetch_releases(org, repo) -> str | None`
   - Call `_gh_api(f"repos/{org}/{repo}/releases?per_page=5")`.
   - Returns a list. If None or empty list, return None.
   - For each release: extract `tag_name`, `name`, `published_at`, `body` (release notes markdown).
   - Title: `# {org}/{repo} — Releases`
   - Each release as `## {tag_name} — {name}\nPublished: {published_at}\n\n{body}`

4. `_fetch_deps(org, repo) -> str | None`
   - Try manifest files in order: `requirements.txt`, `pyproject.toml`, `package.json`.
   - For each: call `_gh_api(f"repos/{org}/{repo}/contents/{filename}")`.
   - On first success: base64-decode `result["content"]`, return it.
   - If all three miss, return None.
   - Title: `# {org}/{repo} — Dependencies`
   - Content: the raw manifest file content under `## {filename}`

5. `_fetch_top_issues(org, repo) -> str | None`
   - Call `_gh_api(f"repos/{org}/{repo}/issues?state=all&sort=reactions&per_page=10&direction=desc")`.
   - Returns a list. If None or empty, return None.
   - Filter out pull requests: skip items where `item.get("pull_request")` is truthy.
   - For each issue: extract `number`, `title`, `state`, `labels` (list of label names), `body` (truncate to first 500 chars), `reactions.total_count`.
   - Title: `# {org}/{repo} — Top Issues (by reactions)`
   - Each issue as `## #{number}: {title}\nState: {state} | Reactions: {count} | Labels: {labels}\n\n{body_truncated}`

**Step 2 — Update ingest_github():**

Replace the single fetch_repo_content() + rag.ainsert() block (lines 173-183) with:

```python
# Fetch all segments
segments: list[tuple[str, str | None]] = [
    ("identity", _fetch_identity(org, repo)),
    ("docs", _fetch_docs(org, repo)),
    ("releases", _fetch_releases(org, repo)),
    ("deps", _fetch_deps(org, repo)),
    ("issues", _fetch_top_issues(org, repo)),
]

# Must have at least identity
if segments[0][1] is None:
    print(f"Error: Could not fetch identity for {org}/{repo}")
    sys.exit(1)

# Compute dedup hash on ALL segment content combined
combined_for_hash = "".join(content for _, content in segments if content)
content_hash = hashlib.md5(combined_for_hash.encode()).hexdigest()[:10]
print(f"Segments fetched: {[name for name, c in segments if c]} (combined hash {content_hash})")

# Insert each segment separately
rag = await get_rag()
for seg_name, seg_content in segments:
    if seg_content:
        print(f"  Inserting segment: {seg_name} ({len(seg_content)} chars)")
        await rag.ainsert(seg_content)
```
