# Quick 260524-tk5 — Diagnose & Fix `long_form` C1 silent 230s hang

## Goal

`POST /api/synthesize {hash:805773ee29, question:"OpenClaw 是什么？它的核心特性、架构和部署方式", mode:"long_form", lang:"zh"}` against the local hydrate currently raises `asyncio.TimeoutError` at the 240s wrapper boundary, falls through to FTS5, and returns ~120 bytes. After this quick:

- HTTP job poll returns `status="done"` with `fallback_used=False`
- `markdown_len > 1000`
- Markdown contains `>=1` inline `![alt](url)` image reference
- Wall-clock end-to-end `< 180s`
- `.uvicorn.log` contains BOTH `before_llm_call` and `after_llm_call` log lines (instrumentation kept in production for diagnostic value)
- Smoke probe (3-of-3 ok) re-passes after the fix

The bug is split into two competing sub-hypotheses:

- **(b1)** SDK request issued, response never arrives — symptom: `before_llm_call` logs once, `after_llm_call` never logs
- **(b2)** Request never issued (LightRAG worker queue / token-count blocker) — symptom: neither log line fires, or `before_llm_call` fires hundreds of seconds after `Final chunks`

T2 disambiguates by reading log timestamps; T3 picks the surgical fix matching whichever branch fired.

### T3 — Surgical fix + verify + commit

**Files:** depends on T2 branch. Default expectation per bug-memo suspicion ranking (b1 highest):

- **b1 (SDK lacks timeout):** Wrap `await rag.aquery(...)` in `kg_synthesize.py` with an inner `asyncio.wait_for(..., timeout=KB_LIGHTRAG_INNER_TIMEOUT)` where `KB_LIGHTRAG_INNER_TIMEOUT = int(os.environ.get("KB_LIGHTRAG_INNER_TIMEOUT", "150"))` is read at module top. 150s budget < 240s wrapper budget so inner timeout fires first with a clean exception that gets caught by the existing `except Exception` retry, triggering the second/third attempt. Each retry sleeps 5s, so worst case 3 × 150 + 2 × 5 = 460s, but in practice attempt 2 should succeed once the worker queue clears.

- **b1-slow (call returns but >180s):** Same fix as b1 — bound the call so we get retries instead of a 230s wait.

- **b2 (token-count or worker-queue blocker before SDK):** Need to inspect the LightRAG `aquery` internals in `lib/llm_complete.py:get_llm_func()` / `lightrag_databricks_provider.make_llm_func()`. THIS IS A THIRD FILE → escalate per scope. Do not fix in this quick; document the b2 evidence and stop.

- **escalate / NEITHER:** Same — stop, document, surface to user.
