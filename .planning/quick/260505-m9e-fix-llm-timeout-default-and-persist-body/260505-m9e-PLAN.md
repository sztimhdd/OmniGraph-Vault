<objective>
Fix two blocking bugs surfaced by the local 5-article cold-graph pilot (quick task 260504-x9l) and Day-1 cron postmortem:

  Bug 1 (LLMT-01): The Vertex Gemini LLM provider default timeout is too tight for 60-image
  articles. With the current default (600s — note: the user spec referenced "900s" but the
  actual hardcoded value in lib/vertex_gemini_complete.py:62 is 600), large articles 100%
  timeout. Bump default to 1800s. Env override behavior is preserved.

  Bug 2 (BODY-01): When scrape_url() succeeds but a downstream stage (classify, ingest)
  fails, the scraped body is held only in process memory and is lost on the next batch run,
  forcing a full re-scrape (~75-90s/article wasted on Apify). Eliminate this SCR-06-class
  data loss by atomically persisting body to articles.body in the main batch loop AFTER a
  successful scrape and BEFORE _classify_full_body() is invoked. Existing _classify_full_body
  internal scrape-on-demand path is left intact for back-compat.

Purpose: Stabilize Day-2/3 KOL baseline so the v3.4 execute gate can lift on schedule.

        def _persist_scraped_body(
            conn: sqlite3.Connection,
            article_id: int,
            scrape: "ScrapeResult",   # forward-ref str so import stays lazy
        ) -> str | None:
            """BODY-01: atomically persist scraped body to articles.body.

            Idempotent: SQL guard `body IS NULL OR length(body) < 500` prevents
            overwriting an already-ingested body (race-safe across batch retries).

            Body source: prefer ScrapeResult.markdown when non-empty, else
            ScrapeResult.content_html. Empty/None bodies are a no-op.

            DB failures are logged at WARNING and swallowed — caller continues.

            Returns the body string on successful UPDATE; None on any failure or no-op.
            """
            try:
                body = (scrape.markdown or "").strip() or (scrape.content_html or "").strip()
                if not body:
                    return None
                conn.execute(
                    "UPDATE articles SET body = ? "
                    "WHERE id = ? AND (body IS NULL OR length(body) < 500)",
                    (body, article_id),
                )
                conn.commit()
                return body
            except Exception as e:  # noqa: BLE001 — never raise into main loop
                logger.warning(
                    "BODY-01 persist failed for article_id=%s: %s",
                    article_id, e,
                )
                return None
