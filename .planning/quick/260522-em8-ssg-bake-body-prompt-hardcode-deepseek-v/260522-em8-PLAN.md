### Task 2 — `lib/translate.py`: SSG-bake body prompt + v4-pro routing

- **Files:** `lib/translate.py`
- **Action (A) — module docstring (lines 16-18):**
  Replace the kb-v2.2-7 R7-mitigation language with SSG-bake disclosure:
  > "Body prompt applies SSG-bake discipline: boilerplate strip, H1 demotion,
  > alt-text enrichment, code fence language inference, paragraph splitting."
- **Action (B) — module-level constant:**
  Add `_BAKE_MODEL = "deepseek-v4-pro"` immediately after the
  `TRANSLATE_BODY_TIMEOUT_S` constant.
- **Action (C) — `_build_body_prompt` body (lines 146-172):**
  Replace docstring + return string with SSG-bake directives:
  - Translate body from `{source_lang}` to `{target_lang}`
  - Heading rule: NEVER output H1 (`#`); demote any H1 in source to H2.
    Top-level sections H2 (`##`), sub-sections H3 (`###`).
  - Strip WeChat boilerplate at body end: 关注公众号, 点赞/在看/分享提示,
    扫码/二维码段落, 转载声明, 作者简介尾段 ("作者：xxx，现任/曾任...").
  - Strip lead filler opening sentences ("今天我们来聊", "大家好",
    "本文将介绍") — first paragraph must open with real content.
  - Image references `![alt](url)`: preserve exact line/paragraph position
    (do NOT batch at end). Translate/generate descriptive alt text in target_lang.
  - Code blocks verbatim, content untranslated. For unlabeled fences (```)
    infer and add a language tag where obvious (python/bash/json/yaml).
  - Long paragraphs (natural break > ~200 chars): split at logical sentence
    boundaries for web readability.
  - Return ONLY the baked markdown. No preamble, no explanation.
