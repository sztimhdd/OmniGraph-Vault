## DECIDE — selected approach

### Decision 1: BL-1 fix — Option A (extend `_translate_one_row` with title path)

- Widen `_select_candidate_rows` WHERE: `body_translated IS NULL OR title_translated IS NULL`
- Branch in `_translate_one_row`:
  - body branch: existing call to `translate_body_with_deepseek_tavily` + UPDATE body_*
  - title branch: call `translate_title_with_deepseek_tavily(title, source_lang=detect)` + UPDATE title_translated
  - Both branches keep their own try/except — failure-safety per RULE-IN-STONE preserved
- Rejected Option B (separate cron file): redundant scaffolding. Rejected Option C (widen-WHERE only): risks re-translating already-good body.
- **Estimated diff: ~30 LoC in `scripts/translate_body_cron.py` + new test in `tests/scripts/test_translate_body_cron.py` (or wherever the existing test of this script lives — discover during execute).**
- **Risk: LOW.** New code path is additive; existing body-only path is preserved.

### Decision 2: BL-2 RSS — defer trigger (4 rows)

- Trigger `systemctl start omnigraph-rss-layer2-classify.service` is mechanically a 0-LoC ops change, BUT the mid-flight constraint forbids further Aliyun SSH.
- **DEFER** the RSS L2 trigger to a follow-up turn after user reviews the BL-1 edit. Once user gives GO + lifts the Aliyun-SSH constraint, RSS L2 can be triggered (`systemctl start`, monitor `journalctl`, then translate cron picks up the newly-'ok' rows on next run).

### Decision 3: BL-2 KOL — SCOPE-OUT

- No standalone KOL-L2-only classifier exists in the repo. Only path to fill the 57 KOL rows' `layer2_verdict` is `batch_ingest_from_spider.py` which also scrapes + LightRAG-ingests, mutating the graph (and contending for LightRAG locks with the systemd ingest timers — explicitly halt-triggered by the original task contract).
- File a BACKLOG note: **"260528-kol-l2-only-classifier"** — write a new ~80-LoC `scripts/kol_layer2_classify_only.py` that reads bodies + classifies + persists L2 verdicts only (no scrape, no graph). Tests required. New cron unit. Out-of-scope for this quick.

### Decision 4: Sequencing

Original contract proposed BL-2-first to maximize translate cron yield. With the SSH-Aliyun-forbidden constraint in effect, the sequence becomes:

1. **Now:** Local code change for BL-1 + tests + commit only. STOP.

This sequencing is **strictly safer** than the original — every Aliyun-side mutation gets explicit user gating.
