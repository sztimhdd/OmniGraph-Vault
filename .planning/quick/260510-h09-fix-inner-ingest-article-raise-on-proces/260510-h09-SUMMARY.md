# Quick 260510-h09 — PROCESSED-gate hot-fix — SUMMARY

## What changed

`ingest_wechat.py` (+79 / -27 LOC):

- Added module-level constants `PROCESSED_VERIFY_MAX_RETRIES = 3` and `PROCESSED_VERIFY_BACKOFF_S = 2.0`.
- Added private async helper `_verify_doc_processed_or_raise(rag, doc_id, *, max_retries, backoff_s)` near `_status_is_processed`. Retries with backoff, raises `RuntimeError` on terminal failure (was: silent `logger.warning(...)` + skip). Final-attempt sleep elided so total wall-clock is bounded at `(max_retries-1) * backoff_s`.
- Replaced the verification region in `ingest_article` with a single `await _verify_doc_processed_or_raise(rag, doc_id); doc_confirmed = True`. The outer `batch_ingest_from_spider.ingest_article` except path catches and marks `ingestions.status='failed'`, so mig 009 retry pool re-queues next cron.
- Updated the inner INSERT to include `source='wechat'`, consistent with mig 008 schema and the outer main-loop INSERT.

## Lifted-constraint test updates (Option A)

The 3 tests below were updated in scope after the in-conversation Option A confirmation. The HARD constraint on these files was lifted because their existing contract ("silent skip") was incompatible with the new contract ("raise on failure"); leaving them broken would have masked regressions.

### `tests/unit/test_text_first_ingest.py` — 3 tests updated

- `test_task08_hook_skips_content_hash_when_doc_absent_from_status` → renamed to `test_task08_hook_raises_runtime_error_when_doc_absent_from_status`. Now asserts `pytest.raises(RuntimeError, match="PROCESSED verification failed")`. Test fixture sqlite DB CREATE TABLE updated to include `source TEXT NOT NULL DEFAULT 'wechat'` (mig 008 parity). `monkeypatch.setattr(ingest_wechat, "PROCESSED_VERIFY_BACKOFF_S", 0.0)` keeps test sub-second.
- `test_task08_hook_skips_content_hash_when_status_not_processed` → renamed to `test_task08_hook_raises_runtime_error_when_status_not_processed`. Mock returns `FAILED` for ALL retry attempts (not just one). Now asserts raise.
- `test_task08_hook_writes_content_hash_when_status_processed` → kept name. Updated CREATE TABLE to include `source` column. Default fixture `_fake_rag` already returns PROCESSED so happy path still works.

### `tests/unit/test_checkpoint_ingest_integration.py` — 6 tests fixed via fixture

The shared `mock_ingest_deps` fixture (line ~78) was updated to add `rag.aget_docs_by_ids = AsyncMock(side_effect=lambda ids: {i: {"status": "PROCESSED"} for i in ids})`. Single-point fix cascades to all 6 affected tests:

### `tests/unit/test_schema_consistency.py` — 1 test, regex made column-aware

`INSERT_RE` previously captured the first quoted literal after `VALUES`, which in the new multi-column INSERT (`(article_id, source, status) VALUES (..., 'wechat', 'ok')`) was `'wechat'` — flagging `'wechat'` as a missing status whitelist entry. Refactored to capture both the column list and the values list, then introduce `_extract_status_literal(col_list, values_list)` which finds `status`'s position in the column list and returns the literal at that position. Returns `None` for non-literal expressions (subqueries, placeholders).
