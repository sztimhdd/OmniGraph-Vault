# Quick Task 260520-m1p (trans-inc): Plan

## Goal

Ship two incremental-translation paths so daily-ingested articles auto-fill the
4 translation columns (`title_translated`, `body_translated`, `translated_lang`,
`translated_at`) in `articles` + `rss_articles` tables. Hermes-only execution;
Aliyun + Databricks consume the translated DB via existing SCP / pull mechanisms.

**Path 1 — inline title translation:** in `batch_ingest_from_spider.py`'s
post-Layer-2 drain loop, after a successful `ingest_article()` and before the
`INSERT INTO ingestions(status='ok')`, translate the article title and UPDATE
the source row. Failure is non-fatal — translation NULL on any error.

**Path 2 — nightly body translation cron:** new script
`scripts/translate_body_cron.py` runs at 03:30 ADT (Hermes-side), translates
~10 articles/day, UPDATEs `body_translated` (preserving any pre-existing
`title_translated`).

Both paths use `lib/llm_deepseek.deepseek_model_complete` (existing client) +
Tavily for terminology context. Tavily augmentation is fail-soft: missing key
or API failure proceeds without web context.

## Context

See `260520-m1p-CONTEXT.md` for codebase-grounded decisions, especially:

- `rss_ingest.py` retired in v3.5 ir-4 — modify only `batch_ingest_from_spider.py`

### Task 2 — `scripts/translate_body_cron.py` nightly cron

**File:** `scripts/translate_body_cron.py` (NEW)

**Behavior:**

1. Parse args: `--dry-run` (boolean), `--limit` (int, default 10)
2. Open DB at `BASE_DIR / "kol_scan.db"` (uses `config.BASE_DIR` so `OMNIGRAPH_BASE_DIR` env override works for local dev)
3. SELECT candidate rows via UNION ALL:

   ```sql
   SELECT id, 'articles' AS table_name, title, body
   FROM articles
   WHERE layer1_verdict='candidate' AND layer2_verdict='ok'
     AND body IS NOT NULL AND body != ''
     AND body_translated IS NULL
   UNION ALL
   SELECT id, 'rss_articles' AS table_name, title, body
   FROM rss_articles
   WHERE layer1_verdict='candidate' AND layer2_verdict='ok'
     AND body IS NOT NULL AND body != ''
     AND body_translated IS NULL
   ORDER BY 1 ASC
   LIMIT ?
   ```

   Note: `ORDER BY layer2_at ASC` cannot apply across UNION easily without subquery wrap; use `ORDER BY 1 ASC` (id ascending = roughly chronological for monotonic IDs) or wrap with subquery if exact `layer2_at` ordering needed — pick subquery wrap with `ORDER BY layer2_at ASC` for spec compliance.
4. For each row:
   - Detect source lang via `detect_source_lang(title or '')`
   - Call `translate_body_with_deepseek_tavily(title, body, source_lang)`
   - On success: `UPDATE {table} SET body_translated = ?, translated_lang = ?, translated_at = ? WHERE id = ?`
     - **DO NOT include `title_translated`** in this UPDATE — preserve any inline-translated title
   - On failure: log warning (id, table, error), continue to next row
   - On `--dry-run`: log "WOULD translate id=X table=Y title=Z[:80]", do not call LLM, do not UPDATE
5. Logging:
   - Configure root logger to emit to both stdout AND `Path(repo_root) / ".scratch" / f"translate-body-cron-{date.today():%Y%m%d}.log"`
   - Format: `%(asctime)s %(levelname)s %(message)s`
   - One line per row attempted (success or fail)
   - Summary line at end: `attempted=N ok=N fail=N skipped=N elapsed=Ns`

**CLI:**

```bash
# Production run
venv/bin/python scripts/translate_body_cron.py

# Local dry-run (no LLM, no UPDATE — verifies SELECT shape)
venv/Scripts/python.exe scripts/translate_body_cron.py --dry-run

# Custom limit

"""Unit tests for lib.translate (260520-trans-inc).

Mocks lib.llm_deepseek.deepseek_model_complete and lib.translate._tavily_search
so no real DeepSeek/Tavily calls. Per CLAUDE.md: tests use unittest.mock,
production-shape assertions.

from __future__ import annotations
import asyncio
import os
os.environ.setdefault("DEEPSEEK_API_KEY", "dummy")  # phase-5 cross-coupling defense

import sqlite3
from pathlib import Path
from unittest.mock import AsyncMock, patch

import pytest

# Lazy import to honor the env-var setdefault above
from lib.translate import (
    detect_source_lang,
    translate_title_with_deepseek_tavily,
)


def test_detect_source_lang_zh():
    assert detect_source_lang("我是一个中文标题") == "zh"
    assert detect_source_lang("AI Agent的实战经验") == "zh"  # mixed but Chinese-dominant


def test_detect_source_lang_en():
    assert detect_source_lang("I am an English title") == "en"
    assert detect_source_lang("LLM Inference at Scale: A Survey") == "en"
    assert detect_source_lang("") == "en"  # empty defaults to en


@pytest.mark.asyncio
async def test_translate_title_fail_returns_none():
    """LLM raise → function swallows error and returns None, never raises."""
    with patch("lib.translate._tavily_search", new=AsyncMock(return_value=[])), \
         patch("lib.translate.deepseek_model_complete",
               new=AsyncMock(side_effect=RuntimeError("mock LLM error"))):
        result = await translate_title_with_deepseek_tavily(
            "Some Title", source_lang="en"
        )
    assert result is None


@pytest.mark.asyncio
async def test_translate_title_returns_dict_on_success():
    """Happy path: LLM returns translated string → function returns dict."""
    with patch("lib.translate._tavily_search", new=AsyncMock(return_value=[])), \
         patch("lib.translate.deepseek_model_complete",
               new=AsyncMock(return_value="中文标题")):
        result = await translate_title_with_deepseek_tavily(
            "English Title", source_lang="en"
        )
    assert result == {"title_translated": "中文标题", "lang": "zh-CN"}


def test_translate_body_skip_already_translated(tmp_path):
    """Cron's SELECT excludes rows where body_translated IS NOT NULL."""
    db_path = tmp_path / "test.db"
    conn = sqlite3.connect(str(db_path))
    # Minimal schema for the cron's SELECT
    conn.executescript("""
        CREATE TABLE articles (
            id INTEGER PRIMARY KEY,
            account_id INTEGER NOT NULL DEFAULT 1,
            title TEXT, body TEXT,
            layer1_verdict TEXT, layer2_verdict TEXT, layer2_at TEXT,
            body_translated TEXT, title_translated TEXT,
            translated_lang TEXT, translated_at TEXT
        );
        CREATE TABLE rss_articles (
            id INTEGER PRIMARY KEY,
            feed_id INTEGER NOT NULL DEFAULT 1,
            title TEXT, body TEXT,
            layer1_verdict TEXT, layer2_verdict TEXT, layer2_at TEXT,
            body_translated TEXT, title_translated TEXT,
            translated_lang TEXT, translated_at TEXT
        );
    """)
    # Two rows: one already translated (should skip), one not (should pick up)
    conn.execute(
        "INSERT INTO articles(id, title, body, layer1_verdict, layer2_verdict, layer2_at, body_translated) "
        "VALUES (1, 't1', 'b1', 'candidate', 'ok', '2026-01-01T00:00:00Z', 'already done')"
    )
    conn.execute(
        "INSERT INTO articles(id, title, body, layer1_verdict, layer2_verdict, layer2_at, body_translated) "
        "VALUES (2, 't2', 'b2', 'candidate', 'ok', '2026-01-02T00:00:00Z', NULL)"
    )
    conn.commit()
    rows = list(conn.execute("""
        SELECT id, 'articles' AS t, title, body FROM articles
         WHERE layer1_verdict='candidate' AND layer2_verdict='ok'
           AND body IS NOT NULL AND body != ''
           AND body_translated IS NULL
        UNION ALL
        SELECT id, 'rss_articles' AS t, title, body FROM rss_articles
         WHERE layer1_verdict='candidate' AND layer2_verdict='ok'
           AND body IS NOT NULL AND body != ''
           AND body_translated IS NULL
    """))
    # Only id=2 (NULL body_translated) should be selected
    assert len(rows) == 1
    assert rows[0][0] == 2
    assert rows[0][1] == "articles"
```
