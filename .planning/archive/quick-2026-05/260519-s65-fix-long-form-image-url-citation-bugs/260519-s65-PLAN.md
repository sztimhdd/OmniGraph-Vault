# Quick 260519-s65 — Plan

## Bug summary (from `.scratch/kb-long-form-bug-report-260519.md`)

- **Bug 1**: long_form markdown emits `![](http://localhost:8765/{hash}/{n}.jpg)` → 404 in browser. KB serves `/static/img/{hash}/{n}.jpg`.
- **Bug 2**: long_form emits `[Entity: X]` style citations instead of `/article/{hash}.html`. `_resolve_sources_from_markdown` regex matches 0 → `sources=[]`, `confidence='no_results'`.

Root cause: `kg_synthesize.py:33-38` `IMAGE_URL_DIRECTIVE` injects `http://localhost:8765/...` instructions into `custom_prompt` for ALL modes; existing long_form template's image instruction is too vague (`![alt](URL)` only) and gets overridden by the more specific `IMAGE_URL_DIRECTIVE`.

## Approach (locked — no rework)

Mirror the FU-1 (`3be03f5`) pattern that fixed the same issue for QA mode:

1. **Tighten `_LONG_FORM_PROMPT_TEMPLATE_ZH/EN`** in `kb/services/synthesize.py`:
   - Change image instruction from `![alt](URL)` → explicit `/static/img/{hash}/{n}.jpg` format
   - Add explicit prohibition: must NOT use `localhost:8765` or absolute URLs
   - Citation `/article/{hash}.html` already present — leave alone

2. **Add `_rewrite_image_urls(markdown: str) -> str` helper**:
   - Regex: `r"https?://[^/\s)]+:8765/"` → `/static/img/`
   - Pure function, no side effects

3. **Apply rewrite in `kb_synthesize` happy path**:
   - Splice between `markdown = response if isinstance(response, str) else ""` (line 489) and `sources = _resolve_sources_from_markdown(markdown)` (line 490)
   - One line: `markdown = _rewrite_image_urls(markdown)`
   - Belt-and-suspenders: prompt instructions are primary; this is the safety net
