<objective>
Fix the daily-ingest cron hang where `batch_ingest_from_spider.py` prints `max-articles cap reached (N) — draining final layer2 queue and stopping.` then refuses to exit. Root cause hypothesis: the existing `_drain_pending_vision_tasks()` (lines 110-153) uses `asyncio.all_tasks()` — a broad scan that captures LightRAG / Cognee / kuzu library tasks alongside the actual Vision workers. Calling `cancel()` on those library tasks (or `gather()`-awaiting them) holds the event loop open indefinitely.

Purpose: replace the broad-scan pattern with a dedicated `_VISION_TASKS` tracking set scoped to `_vision_worker_impl` spawns only. Keeps the existing 120s deadline + cancel-on-timeout semantic intact, but operates on the right task set so cron exits cleanly.

Output: modified `batch_ingest_from_spider.py` + new `tests/unit/test_drain_cap.py` (3 cases). No changes to `image_pipeline.py`, `ingest_wechat.py` Vision logic, `lib/scraper.py`, or `cron_daily_ingest.sh`. The single touch in `ingest_wechat.py` is one line that wraps the existing `asyncio.create_task(_vision_worker_impl(...))` to register the task into the tracking set — the spawn site, not the worker logic.
</objective>

```python
VISION_DRAIN_TIMEOUT = 120.0

async def _drain_pending_vision_tasks() -> None:
    pending = [
        t
        for t in asyncio.all_tasks()           # BROAD SCAN — captures lib tasks
        if t is not asyncio.current_task() and not t.done()
    ]
    if not pending:
        return
    logger.info("Draining %d pending Vision task(s) ...", len(pending))
    try:
        await asyncio.wait_for(
            asyncio.gather(*pending, return_exceptions=True),
            timeout=VISION_DRAIN_TIMEOUT,
        )
        logger.info("Vision tasks drained cleanly")
    except asyncio.TimeoutError:
        still_pending = [t for t in pending if not t.done()]
        logger.warning("Vision drain timeout — %d/%d task(s) still pending (cancelling)", len(still_pending), len(pending))
        for t in still_pending:
            t.cancel()
        if still_pending:
            await asyncio.gather(*still_pending, return_exceptions=True)
```
