# Quick Task 260509-t4i — LightRAG `ainsert` Persistence Contract Tests

## Goal

Add a single new test file `tests/unit/test_ainsert_persistence_contract.py` that codifies
the implicit production contract:

> `await rag.ainsert(content=..., ids=[doc_id])` returns ⇒ `doc_id` MUST appear
> in `kv_store_doc_status.json` with `status="processed"`.

Tests are **regression-isolation only** — no source code changes, no bug fix.
T1 + T2 run by default (mock LLM/embed); T3 (real Vertex) is gated and default-skipped.

## Background — Why This Test Now

### Production evidence (Hermes, 2026-05-09 ADT)

- **`ingestions` table:** `status='ok' AND source='wechat' AND date=2026-05-09` → **41 rows**
- **`kv_store_doc_status.json`:** `status='processed' AND date=2026-05-09` → **15 unique doc_ids**
- **Delta:** **26 silent contract violations** — ingest path reported success but LightRAG
  storage did not finalize.
- **`graph_chunk_entity_relation.graphml` mtime:** frozen at **18:41 ADT** despite ingestions
  continuing to write `ok` rows past 22:00 ADT. The graph stopped growing while the
  ingestions ledger kept marching.

### Why a contract test isolates the layer

The ingest pipeline is layered:

```
batch_ingest_from_spider.py
   → ingest_wechat.py:ingest_wechat
       → rag.ainsert(content, ids=[doc_id])
           → LightRAG internals (chunking → entity extract → KV store flush)
               → kv_store_doc_status.json (filesystem)
```

The **41 vs 15** divergence could live at any of those layers. A 3-tier test isolates which:

| Tier | Scope                              | LLM/embed       | Outcome interpretation                                    | Speed   |
|------|------------------------------------|-----------------|-----------------------------------------------------------|---------|
| T1   | Single doc, mocked LLM/embed       | sync fakes      | If FAIL: LightRAG single-doc path itself broken           | <10s    |
| T2   | Sequential N=7 docs, mocked        | sync fakes      | If FAIL: state leaks across articles in same `LightRAG`   | <60s    |
| T3   | Single doc, real Vertex Gemini     | real            | PASS or hang within 5min (gated, default-skipped)         | 3-5min  |

Reading `kv_store_doc_status.json` **directly off disk** (not via LightRAG API) is
deliberate: the bug surface is the persisted file, exactly what production observed.
