<objective>
Seed the initial code graph by running the `/graphify` skill inside a live Hermes session on the remote PC. This is a human-in-loop, one-shot operation — not automatable because the skill's 7-step pipeline includes an LLM-driven semantic extraction pass (per research §Pitfall 2, there is no `graphify build` CLI).

Purpose: Without `graph.json`, downstream demos (Plans 05) have nothing to query and the weekly cron (Plan 04) has no existing graph to `update`. This plan captures the procedure as a runbook so it can be repeated (e.g., if the graph corrupts) and produces the `graph.json` artifact on remote.

Output: `docs/testing/06-graph-seed-runbook.md` filled with the actual 7-step procedure run, plus the graph artifacts on remote (not committed — live filesystem state).
</objective>
