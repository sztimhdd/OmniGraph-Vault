## Deviations

**1. Multiline reformat in call site reverted to single-line** (deviation Rule 1 — surgical fix to test guard)

- Initial Step 2.4 implementation formatted the call site as a multiline block:
  ```python
  article_budget = _compute_article_budget_s(
      body or "", url=url_d, image_count=image_count_d,
  )
  ```
- This broke `test_drain_layer2_queue_call_site_uses_dynamic_budget` (an existing 2026-05-08 regression-guard test that asserts the literal substring `_compute_article_budget_s(body` is present in the source). The test mirrors the impl token-by-token (CLAUDE.md memory `feedback_test_mirrors_impl.md` is exactly this pattern).
- **Fix:** kept the call single-line with all 3 kwargs inline:
  ```python
  article_budget = _compute_article_budget_s(body or "", url=url_d, image_count=image_count_d)
  ```
- This satisfies both the new D2 semantics AND the existing test guard. No test was modified — Task 3 done criteria preserved ("existing 20 tests in test_timeout_budget.py NOT modified").
- Time cost: 1 retry; 26/27 → 27/27 PASS.

**2. `sqlite3` CLI unavailable on Windows dev** (deviation Rule 3 — environment workaround)

- Plan's `<verify shell="bash">` blocks call the `sqlite3` CLI which is not installed on the Windows dev box (`/usr/bin/bash: line 1: sqlite3: command not found`).
- **Fix:** ran the same semantic checks via Python's built-in `sqlite3` module — copied prod DB to `/tmp/test_mig011_q15.db`, applied `migrations/011_add_image_count.sql` via `conn.executescript()`, then queried `sqlite_master` for the schema text and grep'd for `image_count`.
- Equivalent assertion strength (both rely on `.schema`-style text containing the column name); no semantic loss.
- Production DB never touched.

## Hermes 2026-05-13 Design Review References

D2 was chosen over D1 because:

- **D1 (timeout boundary refactor)** would require moving budget calc inside `ingest_article` post-scrape, which means moving the scrape OUT of the outer `asyncio.wait_for` envelope — adds 2+ failure modes (Apify/CDP/MCP/UA zombie processes need independent cleanup paths).
- **D2 (schema column)** is a single-table migration + single-point write + single function-signature widening — well-trodden path that mirrors mig 008/009/010 idiom.
- Backfill is fast (Hermes estimate: 230 image dirs × `os.listdir` + 2443 UPDATE < 3s).

Hermes review § 4: only WeChat needs the persist hook; RSS scrape path is intentionally untouched because RSS bodies retain `![](remote-url)` markers that the existing `_count_images_in_body` regex already covers.

Hermes review § 5.a: `INTEGER DEFAULT 0` with no CHECK constraint — zero silent-drop risk; non-int values would have been caught by application layer at INSERT time anyway.
